# 食物品質掃描器 - 減脂助手

一個可以掃描食品條碼並即時評估營養價值的網頁應用程式，專為減脂健身人士設計。

## 功能特色

- **條碼掃描**：使用手機相機掃描商品條碼
- **營養分析**：自動從 Open Food Facts 資料庫獲取營養資訊
- **智能評級**：根據蛋白質效率指標 (Ep) 和營養比例進行評分
- **即時建議**：提供購買建議和飲食優化方案
- **手動輸入**：如果掃描不到，可手動輸入營養成分

## 評分標準

### 蛋白質效率指標 (Ep)
```
Ep = 蛋白質(g) / (熱量/100)
```

### 評級系統

| 評級 | Ep 範圍 | 說明 |
|------|---------|------|
| **S+** | Ep ≥ 12 且符合脂肪約束 | 頂級減脂燃料，無腦購買 |
| **A** | 7 ≤ Ep < 12 | 優良選擇，推薦作為正餐 |
| **B** | 5 ≤ Ep < 7 | 需要優化，建議額外補充蛋白質 |
| **F** | Ep < 5 | 不推薦，會影響減脂效果 |

### 營養約束

1. **脂肪約束**：脂肪(g) ≤ 蛋白質(g) / 2
2. **碳水約束**：碳水(g) ≤ 蛋白質(g) × 1.5

## 使用方法

### 方法一：本地使用

1. 下載 `food-scanner.html` 檔案
2. 用瀏覽器（Chrome、Safari、Edge）開啟檔案
3. 允許相機權限
4. 點擊「開始掃描條碼」
5. 對準商品條碼即可

### 方法二：部署到網路

將檔案上傳到任何靜態網站託管服務：

#### 使用 GitHub Pages
1. 在 GitHub 建立新倉庫
2. 上傳 `food-scanner.html`（重新命名為 `index.html`）
3. 在設定中啟用 GitHub Pages
4. 使用手機瀏覽器訪問生成的網址

#### 使用 Netlify/Vercel
1. 拖放檔案到 Netlify Drop 或 Vercel
2. 獲得即時部署的網址
3. 手機即可訪問

### 方法三：手動輸入

如果無法掃描或產品不在資料庫中：
1. 查看商品包裝上的營養標示
2. 在「手動輸入營養成分」區塊填入數值
3. 點擊「計算評級」

## 注意事項

1. **相機權限**：首次使用需允許瀏覽器訪問相機
2. **HTTPS 要求**：網路部署時必須使用 HTTPS（本地檔案不受限）
3. **資料庫限制**：台灣商品在 Open Food Facts 的資料可能不完整，建議使用手動輸入
4. **光線條件**：掃描時確保條碼清晰且光線充足

## 隱私說明

## 常見問題

**Q: 為什麼掃描不到條碼？**
A: 確保光線充足、條碼清晰、相機對焦穩定。如果仍無法掃描，請使用手動輸入。

**Q: 為什麼找不到產品資訊？**
A: Open Food Facts 資料庫可能沒有該產品。建議使用手動輸入功能。

**Q: 可以離線使用嗎？**
A: 手動輸入功能可完全離線使用。條碼掃描需要網路連線查詢資料庫。

**Q: 評分標準可以自訂嗎？**
A: 目前使用固定的減脂標準。如需調整，可修改 HTML 中的 `evaluateFood()` 函數。
